Smart City Reporting System Using AI-Based Image Detection
1. Introduction
1.1 Purpose
The purpose of this document is to describe the functional and technical requirements of the Smart City Reporting System using AI-based Image Detection. The system will allow citizens to report urban problems by uploading images. An Artificial Intelligence model will analyze the image and automatically detect the issue and forward the complaint to the appropriate government department.
The system aims to improve complaint management by introducing:
* automated issue detection
* automatic department routing
* priority-based complaint handling
* duplicate complaint identification
This will help city authorities respond faster to critical issues such as road damage, open manholes, and accidents.
1.2 Scope of the System
The Smart City Reporting System will allow citizens to upload images of city problems through a web interface. The system will use a YOLO object detection model to identify the issue in the image.
After detection, the system will:
1. identify the issue type
2. determine the responsible department
3. assign priority to the complaint
4. check for duplicate complaints
5. store the complaint in the database
6. display complaints in department dashboards
The system will support three major city departments:
* Traffic Department
* Infrastructure Department
* Municipal Department
1.3 Objectives
The main objectives of the system are:
* automate complaint classification using AI
* reduce manual complaint processing
* improve response time for critical issues
* prevent duplicate complaints
* provide structured complaint management for city authorities

4.1 Citizen Complaint Submission

↓

↓
Issue detected

4.2 AI-Based Issue Detection
The system will use the YOLO object detection model to identify the issue present in the uploaded image.
YOLO detects objects and returns:
* object class
* bounding box
* confidence score
Example detection output:
pothole - confidence 0.92
This result becomes the issue type in the complaint system.
4.3 Automatic Department Routing
Each detected issue will be mapped to a department.
Issue TypeDepartmentHelmet ViolationTraffic DepartmentVehicle DetectionTraffic DepartmentPedestrian DetectionTraffic DepartmentAccidentTraffic DepartmentPotholeInfrastructure DepartmentRoad DamageInfrastructure DepartmentRoad CrackInfrastructure DepartmentOpen ManholeMunicipal DepartmentGarbageMunicipal DepartmentWasteMunicipal DepartmentGarbage BinMunicipal DepartmentAfter detection, the complaint is automatically assigned to the corresponding department.
4.4 Priority Assignment System
The system will assign priority based on the detected issue type.
IssuePriorityOpen ManholeCriticalAccidentCriticalPotholeHighRoad DamageHighHelmet ViolationMediumPedestrian DetectionMediumRoad CrackMediumGarbageLowWasteLowGarbage BinLowThis ensures dangerous issues are resolved first.
4.5 Duplicate Complaint Detection
To prevent multiple complaints for the same issue, the system will detect duplicates using three factors:
* Issue Type
* Location
* Time Window
Duplicate rule:
Same issue type
+ Nearby location
+ Submitted within recent time
= Possible duplicate complaint
Example:
Pothole reported at same road within 30 meters
within 3 days
The system will flag such complaints as possible duplicates.

5. AI Model and Dataset
The system will train a YOLOv8 object detection model using multiple datasets.
5.1 Pothole Dataset
Source: Roboflow Public Dataset
Contains images of road surfaces with potholes marked using bounding boxes.
Link:
https://public.roboflow.com/object-detection/pothole
Images: ~665
5.2 Road Damage Dataset (RDD2022)
Large dataset containing road damage images.
Contains:
* potholes
* cracks
* surface damage
Link:
https://github.com/sekilab/RoadDamageDetector
Images: ~47,000
5.3 Garbage Detection Dataset (TACO)
Used for trash detection.
Contains images of:
* litter
* plastic waste
* garbage piles
Link:
https://www.kaggle.com/datasets/vencerlanz09/taco-dataset-yolo-format

5.4 TrashNet Dataset
Used for waste classification.
Link:
https://github.com/garythung/trashnet
Images: ~2500

5.5 Open Manhole Dataset
Used to detect open drainage manholes.
Link:
https://universe.roboflow.com/aibased-solution-for-realtime-detection-of-road-anomalies-d6eay/open-manholes
Images: ~677

11. Conclusion
The Smart City Reporting System aims to improve urban problem reporting by integrating AI-based image detection with automated complaint management. The system addresses several limitations of existing complaint platforms by introducing automated issue detection, department routing, priority-based handling, and duplicate complaint detection.
This system can significantly enhance the efficiency of urban issue management and contribute toward building smarter and more responsive city infrastructure.
